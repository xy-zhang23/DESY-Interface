def plot_config():

Generate input files for Generator and Astra¶

Generator uses only one namelist called Input. Therefore, one just needs to give all the variables and their values when initializing it. The name of variables are case insensitive.

In [13]:
# Define Input module
generator = Generator1(FNAME = 'beam.ini', IPart = 50000, Species = 'electrons', Q_total = -4.,\
                      Ref_Ekin = 0.0e-6, LE = 0.55e-3, dist_pz = 'i',
                      Dist_z = 'p', Lt = 21.5e-3, rt = 2e-3, Cathode = True,\
                      Dist_x = 'r', sig_x = sigma_x, Dist_px = 'g', Nemit_x = 0,\
                      Dist_y = 'r', sig_y = sigma_x, Dist_py = 'g', Nemit_y = 0)

After the initialization, one can update the values of variable by calling the Module.set method. If the specified variable doesn't exist, this method will add the new variable.

Astra has more than one namelists, therefore one needs to initialize them one by one. Similarly, one can update any variable with the Namelist.set method. Note that in some namelist, one variable can be assigned several values, for instance, MaxE in Cavity. In such a case, the variable must be assigned with a list, e.g., MaxE = [60, 12].

In [20]:

Afterwards, one can create an Astra instance and add all the defined modules to it by a list. Finally, call the write method to output files for simulations.

Run Astra within python¶

One can also start Generator and Astra within python, but should add the path to astra/generator to the system variable PATH first.

In the current example, the Track_All is turned off, therefore the simulation ends very soon.
